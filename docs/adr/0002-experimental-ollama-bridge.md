# ADR 0002, 실험적 로컬 Ollama 가이드 브리지

## 상태

채택됨

## 날짜

2026-04-07

## 배경

RewardLab은 현재 epsilon-greedy 다중 슬롯머신 밴딧 단일 레슨을 브라우저 JavaScript에서 실행하는 작은 로컬 학습 애플리케이션입니다. 이번 변경은 사용자가 현재 lesson 안에서 로컬 Ollama를 선택 정책 실험에 연결할 수 있게 해 달라는 요청에서 시작되었습니다.

기존 아키텍처는 다음을 의도적으로 유지하고 있었습니다.

* 기본 학습 루프는 브라우저에서 실행한다.
* Go 서버는 작고 로컬 우선인 호스트로 남는다.
* 일반 목적 JSON API는 제공하지 않는다.

따라서 Ollama 지원은 현재 lesson 경계를 유지하면서 가장 좁은 방식으로 추가해야 했습니다.

## 결정

RewardLab은 기본 epsilon-greedy lesson을 유지하면서, 명시적으로 활성화한 경우에만 동작하는 실험적 로컬 Ollama 가이드 브리지를 추가합니다.

### 선택한 구조

* CLI는 `rewardlab serve --experimental-ollama --ollama-model=<model>` 형태로 실험적 모드를 켤 수 있습니다.
* 서버는 실험적 모드가 켜졌을 때만 `POST /experimental/ollama/guide`를 노출합니다.
* 이 라우트는 현재 lesson 상태에서 다음 arm 선택 제안만 요청합니다.
* 브라우저는 사용자가 직접 opt-in 한 경우에만 이 라우트를 호출합니다.
* 보상 샘플링, true mean, 추정값 갱신, 히스토리 축적은 계속 `web/app.js`가 수행합니다.

### 왜 이 구조를 선택했는가

* 기존 `rewardlab serve` 진입점과 단일 레슨 구조를 유지할 수 있습니다.
* 브라우저가 직접 Ollama에 붙지 않아도 되므로 로컬 구성과 오류 처리를 서버 측에서 좁게 관리할 수 있습니다.
* 일반 목적 모델 API를 만들지 않고 현재 lesson에 필요한 입력과 출력만 다루게 할 수 있습니다.
* “Ollama를 사용한 실험”과 “모델을 실제로 강화학습으로 학습시키는 것”을 구분해 과장 없이 문서화할 수 있습니다.

## 결과

### 장점

* 기본 epsilon-greedy lesson이 그대로 유지됩니다.
* 로컬 Ollama를 현재 lesson 안에서 선택 정책 실험에 사용할 수 있습니다.
* 브리지 표면이 좁아서 현재 저장소 범위를 크게 벗어나지 않습니다.
* Ollama 오류가 나더라도 UI는 기존 epsilon-greedy 경로로 되돌아갈 수 있습니다.

### 트레이드오프

* 기본 v1 아키텍처보다 서버 책임이 조금 늘어납니다.
* guided run은 모델 호출 수를 제한해야 하므로 긴 episode 실험에는 적합하지 않습니다.
* 이 기능은 모델 학습이나 파인튜닝이 아니라 로컬 선택 가이드 실험에 가깝습니다.

## 비목표

이 결정은 다음을 추가하지 않습니다.

* 일반 목적 Ollama API
* 모델 가중치 업데이트나 파인튜닝
* 저장된 실험 기록
* 여러 개의 lesson
* 여러 개의 밴딧 전략

## 후속 기대 사항

앞으로 RewardLab이 진짜 서버 측 시뮬레이션, 일반 목적 API, 또는 모델 학습 파이프라인을 추가하게 된다면, 이 브리지를 조용히 확장하지 말고 새 ADR로 범위와 경계를 다시 결정하세요.
